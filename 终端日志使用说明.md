# 终端日志使用说明

这份文档说明当前项目的终端日志使用方式，包括：

- 如何正常启动 Flutter 开发会话
- 如何在第二个终端里查看彩色日志
- 如何继续使用 `r / R / q / h`
- 如何在代码里继续输出统一格式的日志

## 1. 日常开发推荐方式

推荐使用两个终端同时工作。

### 终端 A：运行 Flutter

先连接模拟器或设备：

```powershell
adb connect 127.0.0.1:5555
```

然后启动项目：

```powershell
flutter run -d 127.0.0.1:5555
```

这个终端主要负责 Flutter 交互命令：

- `r`：Hot Reload
- `R`：Hot Restart
- `q`：退出运行
- `h`：查看全部快捷键

### 终端 B：查看彩色日志

在项目根目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\tool\pretty_logcat.ps1 -DeviceId 127.0.0.1:5555 -ClearBuffer
```

如果你的机器安装了 PowerShell 7，也可以直接用 `pwsh`：

```powershell
pwsh -File .\tool\pretty_logcat.ps1 -DeviceId 127.0.0.1:5555 -ClearBuffer
```

说明：

- `-DeviceId 127.0.0.1:5555`：指定当前设备
- `-ClearBuffer`：启动前先清空旧日志，便于只看本次运行

按 `Ctrl + C` 可以停止日志查看。

## 2. 为什么要用两个终端

如果把 `flutter run` 的输出直接做管道处理，虽然也能着色，但经常会影响 `r / R / q / h` 的交互体验。

所以当前项目推荐：

- 主终端保留给 `flutter run`
- 第二终端专门看日志

这样既能保留 Flutter 的热重载交互，又能获得更清晰的日志显示。

## 3. 当前日志输出格式

项目已经接入统一日志格式，输出大致如下：

```text
12:34:56.123 | SUCCESS | HTTP               | 200 GET https://example.com/api/xxx
12:34:57.104 | WARN    | HOME/LATEST        | Retrying feed request after transient network failure. | error=...
12:34:57.455 | ERROR   | USER/FETCH         | Failed to refresh user profile. | error=...
```

各列含义：

- 时间：日志输出时间
- 级别：`DEBUG / INFO / SUCCESS / WARN / ERROR`
- 模块：例如 `HTTP`、`LOGIN`、`USER/FETCH`
- 内容：具体日志正文

## 4. 颜色规则

`tool/pretty_logcat.ps1` 会按级别着色：

- `ERROR`：红色
- `WARN`：黄色
- `SUCCESS`：绿色
- `INFO`：青色
- 其他：灰色

## 5. 如果只想用一个终端

可以继续直接用原来的命令：

```powershell
flutter run -d 127.0.0.1:5555
```

现在即使不打开第二个终端，日志本身也已经比以前更整齐，至少有：

- 时间戳
- 级别
- 模块名

只是没有第二终端那种彩色区分效果。

## 6. 代码里怎么打日志

不要继续优先使用 `debugPrint(...)`。

推荐统一使用：

```dart
import 'package:tronskins_app/common/logging/app_logger.dart';
```

示例：

```dart
AppLogger.debug('LOGIN', 'Start submit, user=$username');
AppLogger.info('STEAM', 'Token captured.', scope: 'TOKEN_CAPTURED');
AppLogger.success('HTTP', '200 GET https://example.com/api/xxx');
AppLogger.warn('HOME', 'Retrying feed request.', scope: 'LATEST', error: error);
AppLogger.errorLog(
  'USER',
  'Failed to refresh user profile.',
  scope: 'FETCH',
  error: e,
  stackTrace: st,
);
```

建议：

- 第一个参数写模块名，例如 `HTTP`、`LOGIN`、`USER`
- `scope` 用来补充子场景，例如 `FETCH`、`LATEST`、`TOKEN_CAPTURED`
- 错误场景优先传入 `error` 和 `stackTrace`

## 7. 常见问题

### 7.1 `adb` 找不到

如果提示 `adb 未找到`，说明 Android SDK 的 `platform-tools` 没加入环境变量。

先确认下面命令可用：

```powershell
adb version
```

### 7.2 第二个终端没有日志

先检查：

1. `flutter run -d 127.0.0.1:5555` 是否已经启动
2. `adb devices` 是否能看到 `127.0.0.1:5555`
3. 日志脚本里 `-DeviceId` 是否写对

### 7.3 PowerShell 提示“字符串缺少终止符”

这通常不是脚本逻辑错误，而是脚本编码和 `powershell.exe` 的解析兼容问题。

建议按下面顺序处理：

1. 先使用项目里的最新脚本版本重新执行命令
2. 优先使用纯 ASCII 的 `.ps1` 内容
3. 如果机器安装了 PowerShell 7，优先使用：

```powershell
pwsh -File .\tool\pretty_logcat.ps1 -DeviceId 127.0.0.1:5555 -ClearBuffer
```

如果仍然使用 Windows PowerShell 5.x 的 `powershell.exe`，尽量避免在脚本中直接放中文提示文本，或者将脚本保存为 `UTF-8 with BOM`。

### 7.4 日志太多

建议先加 `-ClearBuffer`：

```powershell
powershell -ExecutionPolicy Bypass -File .\tool\pretty_logcat.ps1 -DeviceId 127.0.0.1:5555 -ClearBuffer
```

这样只看当前这次会话的日志，干扰最少。
