# 组件规范

## 组件层级划分

| 层级 | 位置 | 适用场景 |
|------|------|----------|
| 真正通用 Widget | `lib/common/widgets/` | 对话框、Toast、按钮、浮层等，无业务含义，跨所有功能域复用 |
| 功能域组合组件 | `lib/components/<domain>/` | 跨页面复用但绑定特定业务域，如市场卡片、物品属性标签 |
| 页面级私有组件 | `lib/pages/<feature>/_widgets/` | 仅在单个页面内使用，不对外暴露 |

判断依据：
- 多个页面都用 → `lib/components/<domain>/`
- 只有一个页面用 → `lib/pages/<feature>/_widgets/`（目录名加 `_` 前缀表示私有）
- 无业务含义、任何地方都能用 → `lib/common/widgets/`

## 组件文件结构

每个组件一个文件，文件名使用 `snake_case.dart`：

```
lib/components/market/
├── market_item_card.dart       # 市场物品卡片
├── price_tag_widget.dart       # 价格标签
└── wear_badge_widget.dart      # 磨损度徽章

lib/components/game_item/
├── item_quality_tag.dart       # 品质标签
├── item_wear_label.dart        # 磨损度标签
└── item_sticker_row.dart       # 贴纸展示行

lib/common/widgets/
├── app_dialog.dart             # 通用对话框
├── app_toast.dart              # Toast 提示
├── app_button.dart             # 通用按钮
└── app_loading.dart            # 加载指示器
```

## 组件设计规范

### 构造函数
- 所有参数必须明确类型，禁止使用 `dynamic`
- 必填参数使用 `required`，可选参数提供默认值
- 优先使用 `const` 构造函数

```dart
/// 市场物品卡片组件
class MarketItemCard extends StatelessWidget {
  /// 物品数据
  final MarketItem item;

  /// 点击回调
  final VoidCallback? onTap;

  /// 是否显示价格趋势
  final bool showTrend;

  const MarketItemCard({
    super.key,
    required this.item,
    this.onTap,
    this.showTrend = false,
  });

  @override
  Widget build(BuildContext context) { ... }
}
```

### 尺寸与行数限制
- 单个组件文件建议不超过 **300 行**
- `build()` 方法超过 **80 行**必须拆分为私有方法或子 Widget
- 拆分优先使用私有方法（`Widget _buildHeader()`），复杂时抽为独立 Widget 类

```dart
@override
Widget build(BuildContext context) {
  return Column(
    children: [
      _buildHeader(),
      _buildBody(),
      _buildFooter(),
    ],
  );
}

Widget _buildHeader() { ... }
Widget _buildBody() { ... }
Widget _buildFooter() { ... }
```

### 主题与样式
- 颜色统一使用 `AppColors`（`ThemeExtension`），禁止硬编码颜色值
- 字体样式统一使用 `AppTextTheme`，禁止硬编码 `TextStyle`
- 间距使用语义化常量，避免魔法数字

```dart
// ✅ 正确
color: Theme.of(context).extension<AppColors>()!.primary

// ❌ 错误
color: Color(0xFF1976D2)
```

### 国际化
- 所有展示文本必须使用 `'key'.tr`，**禁止硬编码字符串**
- 组件内不得包含翻译键定义，翻译键统一在 `lib/l10n/` 管理

### 图片
- 远程图片统一使用 `CachedNetworkImage`，**禁止**裸 `Image.network`
- 本地资源图片使用 `Image.asset`，路径引用 `assets/` 下已声明的资源

## 物品属性组件（高频复用）

物品属性（品质、磨损度、贴纸）的展示组件封装在 `lib/components/game_item/`，**禁止**在页面层重复实现：

| 组件 | 文件 | 说明 |
|------|------|------|
| 品质标签 | `item_quality_tag.dart` | 根据 quality 显示对应颜色标签 |
| 磨损度标签 | `item_wear_label.dart` | 显示磨损度数值和等级 |
| 贴纸展示 | `item_sticker_row.dart` | 横向展示贴纸列表 |

## 禁止事项

- 禁止在组件中直接调用 API（通过控制器传入数据）
- 禁止在组件中持有或创建 `GetxController` 实例
- 禁止在组件中直接格式化金额（通过 `CurrencyController` 处理后传入）
- 禁止硬编码颜色、字体大小、间距等样式值
- 禁止在 `lib/common/widgets/` 中引入任何业务逻辑或业务模型
