# 编码规范

## Dart 规范

- 所有代码必须使用 Dart，遵循 `flutter_lints` 规则，运行 `flutter analyze` 零警告
- 禁止使用 `dynamic`（除非有明确理由并附注释说明原因）
- 禁止使用裸 `Object`，优先使用泛型约束
- 所有公开 API（类、方法、属性）必须添加 `///` 文档注释，使用中文说明
- 优先使用 `const` 构造函数，减少不必要的 Widget 重建
- 禁止在 `build()` 方法中执行业务逻辑或网络请求

## 命名规范

| 类型 | 规则 | 示例 |
|------|------|------|
| 文件 / 目录 | `snake_case` | `market_detail_page.dart` |
| 变量 / 函数 | `camelCase` | `itemList`, `fetchData` |
| 常量 | `camelCase`（Dart 惯例） | `apiTimeout`, `maxRetryCount` |
| 类 / 枚举 | `PascalCase` | `MarketItem`, `OrderStatus` |
| 控制器 | `FeatureController` | `MarketController` |
| Binding | `FeatureBinding` | `MarketBinding` |
| 页面 Widget | `FeaturePage` | `MarketDetailPage` |
| 可复用组件 | `FeatureWidget` / 语义名 | `PriceTagWidget`, `ItemCard` |
| 路由常量 | `SCREAMING_SNAKE_CASE`（`Routers` 静态成员） | `Routers.MARKET_DETAIL` |
| 响应式变量 | `RxType` + `.obs` | `final count = 0.obs` |
| 私有成员 | `_camelCase` | `_isLoading`, `_fetchItems` |

## 业务函数命名

- 事件响应（绑定到 UI 回调）：`onXxx`，如 `onSubmit`、`onItemTap`
- 内部处理逻辑：`handleXxx`，如 `handleDelete`、`handlePageChange`
- 数据获取：`fetchXxx` / `loadXxx`，如 `fetchMarketList`、`loadUserInfo`
- 数据提交：`submitXxx` / `saveXxx`，如 `submitOrder`、`saveSettings`
- 状态切换：`toggleXxx`，如 `toggleFavorite`
- 格式化 / 转换：`formatXxx` / `parseXxx`，如 `formatPrice`、`parseDate`

## 注释规范

- 公开类和方法必须有 `///` 文档注释，使用中文
- 复杂逻辑块使用 `//` 行注释说明意图，使用中文
- 禁止无意义注释（如 `// 获取数据` 紧跟 `getData()`）
- 临时代码或待处理项用 `// TODO: 说明` 标记

```dart
/// 市场列表控制器，负责管理皮肤列表的加载、筛选和排序
class MarketController extends GetxController {
  /// 当前皮肤列表
  final items = <MarketItem>[].obs;

  /// 加载市场列表数据
  Future<void> fetchMarketList() async { ... }

  /// 处理筛选条件变更
  void handleFilterChange(FilterParams params) { ... }

  /// 用户点击购买按钮时触发
  void onBuyTap(MarketItem item) { ... }
}
```

## Widget 规范

- 页面 Widget 保持"薄"：只读取控制器数据、响应用户事件，不含业务逻辑
- 超过 **80 行**的 `build()` 方法必须拆分为私有子 Widget 方法或独立 Widget 类
- 列表项、卡片等可复用 UI 抽取到 `lib/components/` 对应功能域目录
- 禁止在页面层直接处理金额格式化，统一通过 `CurrencyController`
- 所有 UI 字符串必须使用翻译键（`'key'.tr`），**禁止硬编码文本**
- 远程图片统一使用 `CachedNetworkImage`，**禁止**裸 `Image.network`

## 模型规范

- API 响应模型使用 `json_annotation` + `json_serializable`，生成 `.g.dart`
- 不可变值对象使用 `freezed`，生成 `.freezed.dart`
- 生成文件（`*.g.dart`、`*.freezed.dart`）**必须提交到仓库**
- 修改任何带注解的模型后，必须重新运行：
  ```bash
  dart run build_runner build --delete-conflicting-outputs
  ```

## 错误处理规范

- 网络请求统一在 `HttpHelper` 拦截器层处理通用错误，业务层只处理业务异常
- 禁止空 `catch` 块，至少记录日志：`logger.e('描述', error: e, stackTrace: s)`
- 用户可见的错误提示通过 `Get.snackbar` 或统一 Toast 组件展示，禁止直接 `print`
