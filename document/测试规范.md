# 测试规范

## 测试框架

| 工具 | 用途 |
|------|------|
| `flutter_test`（内置） | 单元测试、Widget 测试 |
| `flutter analyze` | 静态分析，零警告才算通过 |
| `flutter test` | 运行所有测试 |

项目**不引入**额外 mock 框架，使用手写 Fake 类替代（见下文）。

## 测试目录结构

测试目录镜像 `lib/` 结构：

```
test/
├── widget_test.dart              # 应用启动冒烟测试
└── controllers/
    └── auth/
        └── steam_session_controller_test.dart
```

新增测试文件路径规则：`test/<同 lib/ 下的相对路径>_test.dart`

示例：
- `lib/controllers/wallet/wallet_controller.dart` → `test/controllers/wallet/wallet_controller_test.dart`
- `lib/common/utils/price_utils.dart` → `test/common/utils/price_utils_test.dart`

## 必须编写测试的场景

| 场景 | 优先级 |
|------|--------|
| 资金相关逻辑（充值、提现、结算、余额计算） | 必须 |
| Steam 认证流程（登录、Session、2FA） | 必须 |
| 工具函数（`lib/common/utils/`） | 必须 |
| 控制器核心状态流转（加载、成功、失败） | 必须 |
| API 响应模型的序列化/反序列化 | 推荐 |
| 通用 Widget 的渲染逻辑 | 推荐 |

## 测试文件结构

```dart
import 'package:flutter_test/flutter_test.dart';
import 'package:get/get.dart';

void main() {
  // GetX 测试模式，防止路由/依赖注入干扰
  setUpAll(() {
    Get.testMode = true;
  });

  // 每个测试后清理 GetX 注册
  tearDown(() {
    Get.reset();
  });

  group('WalletController', () {
    test('初始状态下余额为 0', () {
      // Arrange
      final controller = WalletController(api: _FakeWalletApi());

      // Act（onInit 已在构造后自动调用）

      // Assert
      expect(controller.isLoadingUser.value, isFalse);
    });

    test('loadFundFlows 加载成功后填充列表', () async {
      // Arrange
      final api = _FakeWalletApi()..fundFlows = [_fakeFundFlow()];
      final controller = WalletController(api: api);

      // Act
      await controller.loadFundFlows(reset: true);

      // Assert
      expect(controller.fundFlows, hasLength(1));
      expect(controller.isLoadingFundFlows.value, isFalse);
    });
  });
}
```

## Fake 类规范（替代 Mock）

项目统一使用手写 Fake 类，**不引入** `mockito` 或 `mocktail`：

```dart
/// Fake API，用于隔离网络依赖
class _FakeWalletApi extends ApiWalletServer {
  // 预设返回数据，测试中按需覆盖
  List<WalletFundFlowItem> fundFlows = [];
  bool shouldFail = false;

  @override
  Future<BaseHttpResponse<WalletFundFlowData>> fundChangesList({
    required int page,
    required int pageSize,
  }) async {
    if (shouldFail) throw Exception('网络错误');
    return BaseHttpResponse(
      code: 200,
      message: '',
      datas: WalletFundFlowData(list: fundFlows, pager: null),
    );
  }
}
```

Fake 类命名规则：`_Fake<ClassName>`，以 `_` 开头表示文件私有。

## Widget 测试

通用 Widget 和关键页面组件使用 `testWidgets` 验证渲染逻辑：

```dart
import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';
import 'package:get/get.dart';

void main() {
  testWidgets('PriceTagWidget 正确显示价格文本', (tester) async {
    await tester.pumpWidget(
      const GetMaterialApp(
        home: Scaffold(
          body: PriceTagWidget(price: '\$12.50'),
        ),
      ),
    );

    expect(find.text('\$12.50'), findsOneWidget);
  });

  testWidgets('AppLoading 显示加载指示器', (tester) async {
    await tester.pumpWidget(
      const GetMaterialApp(home: Scaffold(body: AppLoading())),
    );

    expect(find.byType(CircularProgressIndicator), findsOneWidget);
  });
}
```

Widget 测试注意事项：
- 必须用 `GetMaterialApp` 包裹，否则 `'key'.tr` 会抛出异常
- 调用 `tester.pump()` 触发一帧，`tester.pumpAndSettle()` 等待所有动画完成
- 验证文本用 `find.text()`，验证组件类型用 `find.byType()`

## 控制器测试要点

控制器通过构造函数注入依赖，便于测试时替换：

```dart
// ✅ 控制器设计：支持依赖注入
class WalletController extends GetxController {
  WalletController({ApiWalletServer? api}) : _api = api ?? ApiWalletServer();
  final ApiWalletServer _api;
}

// ✅ 测试中注入 Fake
final controller = WalletController(api: _FakeWalletApi());
```

测试控制器时需注意：
- `onInit` 在 `GetxController` 实例化后**不会自动调用**，需手动调用或通过 `Get.put` 触发
- 使用 `Get.testMode = true` 防止路由跳转抛出异常
- 异步操作需 `await` 或配合 `Future.delayed` 等待完成

```dart
test('切换游戏后自动刷新数据', () async {
  final api = _FakeMarketApi();
  final controller = MarketController(api: api);
  controller.onInit(); // 手动触发初始化

  // 模拟游戏切换
  GlobalGameController.ensureInstance().currentAppId.value = 570;
  await Future<void>.delayed(const Duration(milliseconds: 10));

  expect(api.fetchCount, greaterThan(1));
});
```

## 异步测试

```dart
// 等待异步操作完成
test('提现成功后刷新余额', () async {
  final controller = WalletController(api: _FakeWalletApi());

  final result = await controller.submitWithdraw(amount: 10.0, account: 'addr');

  expect(result, isTrue);
  expect(controller.isLoadingUser.value, isFalse);
});

// 等待定时器/轮询（使用 fake async 或 Future.delayed）
test('轮询超时后停止', () async {
  final controller = SteamSessionController(
    pollInterval: const Duration(milliseconds: 20),
    maxPollAttempts: 2,
  );
  await controller.startLogin();
  await Future<void>.delayed(const Duration(milliseconds: 50));

  expect(controller.isAwaitingVerification.value, isFalse);
  controller.onClose();
});
```

## 边界条件测试

关键业务逻辑必须覆盖边界条件：

```dart
group('CurrencyController.format', () {
  test('负数返回零值字符串', () {
    expect(controller.format(-1.0), equals(r'$ 0'));
  });

  test('NaN 返回零值字符串', () {
    expect(controller.format(double.nan), equals(r'$ 0'));
  });

  test('零值正常格式化', () {
    expect(controller.format(0.0), equals(r'$ 0.00'));
  });
});
```

## 静态分析门禁（NON-NEGOTIABLE）

所有代码提交前必须通过：

```bash
flutter analyze   # 零警告，零错误
flutter test      # 所有测试通过
```

单行抑制 lint 规则时必须附说明：

```dart
// ignore: non_constant_identifier_names  — 路由常量历史命名，保持一致性
static String MARKET = '/market';
```

**禁止**使用 `// ignore_for_file:` 抑制整个文件的规则。

## 禁止事项

- 禁止引入 `mockito`、`mocktail` 等 mock 框架，统一使用手写 Fake 类
- 禁止在测试中直接调用真实网络请求
- 禁止测试文件中硬编码生产环境 URL 或密钥
- 禁止跳过失败测试（`skip: true`）而不附说明原因
- 禁止在测试中使用 `sleep` 或过长的 `Future.delayed`（超过 500ms）
- 禁止提交 `flutter analyze` 有警告的代码
