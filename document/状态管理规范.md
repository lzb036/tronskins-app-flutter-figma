# 状态管理规范

## 控制器分类

项目中存在两类控制器，职责和注册方式完全不同：

| 类型 | 基类 | 位置 | 注册方式 | 生命周期 |
|------|------|------|----------|----------|
| 全局永久控制器 | `GetxController` / `GetxService` | `lib/common/hooks/` | `main()` 中 `Get.put(..., permanent: true)` | 应用全生命周期 |
| 路由级控制器 | `GetxController` | `lib/controllers/` | 对应 `Binding` 的 `Get.lazyPut` | 路由生命周期 |

## 全局永久控制器

位于 `lib/common/hooks/`，在 `main()` 中注册，全应用共享：

| 控制器 | 文件 | 职责 |
|--------|------|------|
| `GlobalGameController` | `hooks/game/` | 当前游戏 appId，所有 API 请求必须携带 |
| `CurrencyController` | `hooks/currency/` | 货币格式化，页面层禁止直接处理金额 |
| `UseTheme` | `hooks/theme/` | 主题切换，持久化到 `GetStorage('theme')` |
| `UseLocale` | `hooks/locale/` | 语言切换，持久化到 `GetStorage('language')` |

### 访问全局控制器

```dart
// 推荐：通过 Get.find 获取
final currency = Get.find<CurrencyController>();

// GlobalGameController 提供了 ensureInstance 工厂方法
final game = GlobalGameController.ensureInstance();

// CurrencyController 提供了静态 to getter
final currency = CurrencyController.to;
```

**禁止**在路由 Binding 中重复注册全局永久控制器。

### 控制器模板

```dart
/// 市场列表控制器，负责管理皮肤列表的加载、筛选和排序
class MarketController extends GetxController {
  // 依赖注入（通过构造函数，便于测试）
  MarketController({ApiMarketServer? api}) : _api = api ?? ApiMarketServer();

  final ApiMarketServer _api;

  // 响应式状态用 .obs
  final RxList<MarketItem> items = <MarketItem>[].obs;
  final RxBool isLoading = false.obs;

  // 非响应式私有状态
  int _page = 1;
  bool _hasMore = true;

  // Worker 引用，onClose 时必须 dispose
  Worker? _gameWorker;

  @override
  void onInit() {
    super.onInit();
    // 监听全局游戏切换
    _gameWorker = ever<int>(
      GlobalGameController.ensureInstance().currentAppId,
      (_) => fetchMarketList(reset: true),
    );
    fetchMarketList(reset: true);
  }

  @override
  void onClose() {
    _gameWorker?.dispose();
    super.onClose();
  }

  /// 加载市场列表
  Future<void> fetchMarketList({bool reset = false}) async {
    if (isLoading.value) return;
    isLoading.value = true;
    try {
      final appId = GlobalGameController.ensureInstance().appId;
      // ... 请求逻辑
    } catch (e, s) {
      logger.e('获取市场列表失败', error: e, stackTrace: s);
    } finally {
      isLoading.value = false;
    }
  }
}
```

## Binding 规范

每个需要独立控制器生命周期的路由必须配置对应 Binding：

```dart
// lib/bindings/market/market_binding.dart
class MarketBinding extends Bindings {
  @override
  void dependencies() {
    Get.lazyPut<MarketController>(() => MarketController());
  }
}
```

**规则**：
- 只使用 `Get.lazyPut`，**禁止** `Get.put`
- 一个 Binding 可注册多个控制器（同一路由需要的所有控制器）
- **禁止**在 Binding 中注册全局永久控制器

## 响应式状态规范

### 声明

```dart
// 基本类型
final count = 0.obs;
final name = ''.obs;
final isLoading = false.obs;

// 可空类型
final user = Rx<UserInfo?>(null);
final Rxn<UserInfo> user = Rxn<UserInfo>();  // 等价写法

// 集合
final items = <MarketItem>[].obs;
final map = <String, dynamic>{}.obs;
```

### 读取与更新

```dart
// 读取
final value = count.value;

// 更新
count.value = 1;
isLoading.value = true;

// 集合操作（推荐，触发响应式更新）
items.assignAll(newList);   // 替换全部
items.addAll(moreItems);    // 追加
items.clear();              // 清空
items.removeWhere((e) => e.id == id);
```

### UI 中订阅

```dart
// Obx：最小粒度重建，推荐
Obx(() => Text(controller.count.value.toString()))

// GetX：可同时访问控制器
GetX<MarketController>(
  builder: (ctrl) => Text(ctrl.items.length.toString()),
)

// GetBuilder：非响应式，手动 update() 触发
GetBuilder<MarketController>(
  builder: (ctrl) => Text(ctrl.someValue),
)
```

## Worker（响应式监听）

```dart
// ever：每次变化都触发
_worker = ever(someRx, (value) => handleChange(value));

// once：只触发一次
once(someRx, (value) => doOnce(value));

// debounce：防抖（用户输入场景）
debounce(searchQuery, (_) => search(), time: const Duration(milliseconds: 300));

// interval：节流
interval(someRx, (_) => refresh(), time: const Duration(seconds: 1));
```

**必须**在 `onClose()` 中 dispose Worker，防止内存泄漏：

```dart
@override
void onClose() {
  _worker?.dispose();
  super.onClose();
}
```

## 跨控制器通信

**禁止**跨功能域直接引用另一个域的控制器，通过事件总线通信：

```dart
// 发送事件（lib/common/events/app_events.dart）
AppEvents.triggerUserLogout();

// 监听事件（在 onInit 中注册 Worker）
_logoutWorker = ever(AppEvents.userLogoutEvent, (_) => clearSession());
```

同域内可以通过 `Get.find` 访问：

```dart
// 同域内访问（如 WalletController 访问 UserController）
if (Get.isRegistered<UserController>()) {
  await Get.find<UserController>().fetchUserData(showLoading: false);
}
```

## 多游戏支持（NON-NEGOTIABLE）

所有涉及游戏数据的控制器必须监听 `GlobalGameController.currentAppId` 变化：

```dart
@override
void onInit() {
  super.onInit();
  appId.value = GlobalGameController.ensureInstance().currentAppId.value;
  _gameWorker = ever<int>(
    GlobalGameController.ensureInstance().currentAppId,
    (nextAppId) {
      if (nextAppId == appId.value) return;
      appId.value = nextAppId;
      refreshAll();  // 切换游戏后刷新数据
    },
  );
  refreshAll();
}
```

**禁止**在 API 函数内部直接调用 `GlobalGameController`，appId 由控制器传入。

## 货币格式化（NON-NEGOTIABLE）

**禁止**在页面或控制器中直接格式化金额，统一通过 `CurrencyController`：

```dart
// ✅ 正确
final currency = CurrencyController.to;
Text(currency.format(item.price))       // 按当前货币格式化
Text(currency.formatUsd(item.price))    // 固定 USD 显示

// ❌ 错误
Text('\$${item.price.toStringAsFixed(2)}')  // 禁止硬编码格式化
```

## 错误处理规范

```dart
// ✅ 正确：只记录日志，通用错误由拦截器处理
Future<void> fetchData() async {
  if (isLoading.value) return;
  isLoading.value = true;
  try {
    items.assignAll(await _api.getList());
  } catch (e, s) {
    logger.e('获取数据失败', error: e, stackTrace: s);
  } finally {
    isLoading.value = false;
  }
}

// ❌ 错误：重复弹出错误提示（拦截器已处理）
} catch (e) {
  Get.snackbar('错误', e.toString());
}
```

## 禁止事项

- 禁止在控制器中持有 Widget 引用
- 禁止在页面中手动 `Get.put` 路由级控制器（必须通过 Binding）
- 禁止在路由 Binding 中重复注册全局永久控制器
- 禁止跨功能域直接 `Get.find` 另一个域的控制器（通过事件总线通信）
- 禁止在 `build()` 方法中执行业务逻辑或网络请求
- 禁止空 `catch` 块，至少记录日志
- 禁止在页面层直接格式化金额，统一通过 `CurrencyController`
- 禁止在 API 函数内部调用 `GlobalGameController`，appId 由控制器传入
