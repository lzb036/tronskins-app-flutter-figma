# 路由规范

## 文件结构

```
lib/routes/
├── app_routes.dart   # Routers 类，静态字符串路由常量
└── index.dart        # RoutersConfig.list → List<GetPage>
```

## 路由常量规范（NON-NEGOTIABLE）

所有路由常量定义在 `lib/routes/app_routes.dart` 的 `Routers` 类中：

- 命名规则：`SCREAMING_SNAKE_CASE`
- 路径规则：`kebab-case`，按功能域分层，如 `/wallet/recharge`
- 每个常量必须加 `// ignore: non_constant_identifier_names` 抑制 lint 警告

```dart
class Routers {
  // ignore: non_constant_identifier_names
  static String WALLET = '/wallet';
  // ignore: non_constant_identifier_names
  static String WALLET_RECHARGE = '/wallet/recharge';
  // ignore: non_constant_identifier_names
  static String WALLET_WITHDRAW = '/wallet/withdraw';
}
```

## GetPage 注册规范

所有路由在 `lib/routes/index.dart` 的 `RoutersConfig.list` 中注册：

```dart
class RoutersConfig {
  static List<GetPage> list = [
    // 带 Binding 的路由（需要控制器注入）
    GetPage(
      name: Routers.HOME,
      page: () => NavBarPage(),
      binding: UserBinding(),
      transition: Transition.fade,
    ),
    // 普通路由
    GetPage(name: Routers.WALLET, page: () => WalletPage()),
    // 带参数的路由（通过 Get.arguments 传参）
    GetPage(name: Routers.SHOP_ORDER_DETAIL, page: () => ShopOrderDetailPage()),
  ];
}
```

### Binding 使用规则

- 需要独立控制器生命周期的路由**必须**配置对应 `Binding`
- Binding 只通过 `Get.lazyPut` 注册控制器，**禁止** `Get.put`
- 全局永久控制器（`GlobalGameController` 等）已在 `main()` 注册，路由 Binding 中**禁止**重复注册

### 转场动画

- 默认不指定（使用平台默认动画）
- 首页（NavBar）使用 `Transition.fade`
- 特殊场景可按需指定，保持同一功能域内风格一致

## 路由导航规范

### 基本导航

```dart
// 跳转到新页面
Get.toNamed(Routers.WALLET);

// 跳转并替换当前页面（不可返回）
Get.offNamed(Routers.LOGIN);

// 跳转并清空所有历史（登录后跳首页）
Get.offAllNamed(Routers.HOME);

// 返回上一页
Get.back();

// 返回并携带结果
Get.back(result: data);
```

### 参数传递

```dart
// 传递参数（接收方通过 Get.arguments 获取）
Get.toNamed(Routers.SHOP_ORDER_DETAIL, arguments: {'orderId': '123'});

// 传递路由参数（接收方通过 Get.parameters 获取）
Get.toNamed('${Routers.MARKET_DETAIL}?id=123');
```

**禁止**在路由路径中硬编码参数，统一使用 `arguments` 传递复杂对象。

## 新增路由检查清单

新增一个功能路由，必须同时完成：

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `lib/routes/app_routes.dart` | 添加 `Routers.FEATURE_NAME` 常量 |
| 2 | `lib/routes/index.dart` | 在 `RoutersConfig.list` 中注册 `GetPage` |
| 3 | `lib/bindings/<domain>/` | 如需控制器注入，创建对应 Binding |
| 4 | `lib/pages/<domain>/` | 创建对应页面 Widget |

缺少任何一步均视为不完整实现。

## 禁止事项

- 禁止在页面或控制器中硬编码路由路径字符串，必须使用 `Routers.XXX` 常量
- 禁止在 Binding 中使用 `Get.put`，只允许 `Get.lazyPut`
- 禁止在路由 Binding 中重复注册全局永久控制器
- 禁止跳过 `RoutersConfig.list` 直接使用 `Get.to(() => SomePage())`（无法享受 Binding 和转场配置）
- 禁止在 `app_routes.dart` 以外的地方定义路由路径字符串
